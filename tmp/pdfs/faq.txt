AI Agent Project – FAQs 
 
This FAQ is an authoritative reference for students implementing the AI Agent Project. All questions are 
mandatory reading before starting the project. 
1. Language, Technology & Tooling 
Q1. Can we use TypeScript instead of JavaScript? 
Yes. Both JavaScript and TypeScript are allowed. If TypeScript is chosen, the entire backend must be 
written consistently in TypeScript. Mixing JS and TS in the backend is not allowed. In case you are using 
N8N, Claude Code to conduct the tasks from a GUI, make sure the prompt reflects the desired outcome. 
Q2. What backend stack is mandatory? 
No particular technology is mandatory. Node.js with Express.js is preferred for code-based agent 
development. In that case, AI agent logic must be implemented using LangChain (JavaScript or 
TypeScript). But for N8N, Claude Code or similar GUI-based tools, the backend will be embedded and the 
students must focus on the node flow or sequence flow to ensure desired outcome. 
Q3. Which IDE should be used? 
WebStorm is preferred as used in class. VS Code, Cursor, or any standard IDE is fully acceptable and will 
not impact grading. For GUI-based tools like N8N, the default editor is fine.  
Q4. Is React with Vite mandatory? 
No. React + Vite is optional. Backend-only projects tested via API calls (Postman / curl) are acceptable. 
2. TypeScript Setup (If You Choose TypeScript) 
Q5. What packages must be installed for TypeScript projects? 
The following packages are mandatory for TypeScript-based projects: 
npm install -D typescript ts-node nodemon @types/node @types/express 
Q6. Do we need a tsconfig.json file? 
Yes. A tsconfig.json file is mandatory. Recommended minimum configuration: 
{ 
  "compilerOptions": { 
    "target": "ES2020", 
    "module": "CommonJS", 
    "rootDir": "src", 
    "outDir": "dist", 
    "strict": true,

Q7. What should be the main entry file in TypeScript? 
Use src/index.ts as the application entry point. Do not use index.js for TypeScript projects. 
3. LLMs, Tools & Agent Logic 
Q8. Which LLMs are allowed? 
OpenAI is recommended. Hugging Face models or Ollama are allowed with conditions (see Section 6). 
Q9. Is tool usage mandatory? 
Yes. Every agent must use at least one tool (e.g., Tavily search, API calls, calculation logic). A project 
without tools will not be considered an agent. For N8N, for example, there are a long list of supported 
tools that students can pick from.  
Q10. How do we verify the agent is behaving correctly? 
The agent must reason clearly, call tools only when needed, avoid hallucinating tool outputs, and respond 
consistently across tests. 
4. Correct Flow of Work  
Q11. What is the correct development sequence? 
Students should follow this order if code-based AI agent is being developed: 
• 
Initialize Node.js project 
• 
Install packages (Express + dotenv + LangChain + LLM SDK + tools)  
• 
Set up Express backend (server runs on a port) 
• 
Create /chat endpoint (initially return a stub response) 
• 
Integrate LLM credentials/config (.env, model init) 
• 
Define agent logic using LangChain 
• 
Register tools (at least one) 
• 
Test using Postman/curl (all test cases) 
• 
Add frontend (optional) 
For GUI-based AI agent development tools like N8N, the flow starts and stops at the N8N editor with 
intermediate step out to configure the tools and channels for API keys. 
5. Environment, Hosting & Demo 
Q12. Do we need cloud hosting or a domain? 
No permanent hosting or domain is required. Running locally and exposing via a temporary public URL is 
sufficient. For schedule-based AI agents, this is not required.

Q13. What must work on demo day? 
The agent must run without errors and be accessible via a public URL/ output tool/ app as applicable 
based on the nature of the AI agent. The team must be able to explain the full agent flow. 
6. Using Hugging Face Models or Ollama (Special Instructions) 
Q14. Can we use Hugging Face models for this project? 
Yes, Hugging Face models are allowed provided: (1) You can clearly demonstrate reasoning, (2) Tool usage 
remains external to the model, (3) You can explain latency, prompt format, and limitations. Do not use 
pre-built chat UIs as your agent. 
Q15. Can we use Ollama for local LLM execution? 
Yes, Ollama is allowed for local LLM execution. However, the agent must still follow LangChain-based 
control flow. Direct prompt-to-response scripts without tools or orchestration are not acceptable. 
Q16. What are the DOs and DON’Ts for Hugging Face / Ollama? 
DOs: 
• Use LangChain to orchestrate reasoning and tool calls 
• Treat the LLM as one component, not the system itself 
• Be able to explain model choice and limitations 
DON’Ts: 
• Do not use turnkey chatbot demos 
• Do not bypass tool usage 
• Do not submit UI-only or notebook-only solutions 
7. Laptop Requirements, Cost & Licensing 
Q17. What laptop configuration is sufficient? 
Any modern laptop with at least 8 GB RAM and stable internet is sufficient. GPU is not required. 
Q18. Are there any licensing or cost issues? 
Node.js, TypeScript, Express, LangChain are free. OpenAI or other hosted LLM usage may incur minimal 
API costs. 
8. Grading & Evaluation 
Q19. Will every team member be questioned? 
Yes. Each team member must be able to explain the system architecture, agent reasoning, and tool usage. 
Q20. What matters most for grading? 
Priority is given to agent reasoning quality, architecture clarity, live demo reliability, and explainability. UI 
polish is least important.
